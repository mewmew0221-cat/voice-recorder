# 手動優化提示詞（用外部高階模型）

桌面端的 🧬 一鍵優化走的是你 GAS 設定的同一個模型（預設 `gemini-2.0-flash-lite`）。
當你想針對某個重要分類「精雕」、改用更強的模型（Claude Opus、Gemini 2.5 Pro、GPT 等）時，
可用這套手動流程：把編輯回饋拉出來，貼到高階模型跑，再把結果貼回 `prompts` 表。

**完全不需要改任何程式碼。**

---

## ⚠️ 先讀：資料外流提醒

匯出的 `feedback` 內含 `ai_output` / `user_edited`，也就是**整理後的病歷文字**。
把它貼到外部 AI 服務 = 醫療資料離開院內。請注意：

- **個人 / 免費版**的 Claude.ai、ChatGPT 等，對話可能被用於模型訓練。
- 建議擇一：
  1. 用**不會拿去訓練**的方案（API、企業/團隊版、或在設定中關閉訓練）；
  2. 或貼上前先**去識別化**（移除姓名、病歷號、床號等可識別資訊）。
- 若無法接受，就改用 🧬 一鍵優化（資料只經過你自己的 API Key，不進網頁版服務）。

> 註：日常整理本來就會把文字送到 Gemini/Claude **API**；此處的差別只是「改送到網頁版服務」，
> 而網頁版個人方案的訓練政策通常比 API 寬鬆，故特別提醒。

---

## 步驟

### 1. 匯出該分類的編輯回饋

瀏覽器開（把網址、TOKEN、分類換成你的）：

```
你的GAS網址/exec?action=feedback&category=ward_round&token=你的TOKEN
```

- `category` 可省略 → 匯出全部分類
- 分類 key 對照：`ward_round`（查房）、`admission`（住院病程）、`meeting`（會議）、
  `important_conversation`（重要對話）、`idea_note`（靈感）、`text_note`（文字筆記）

畫面會出現一段 JSON，**整段複製**。重點欄位：
- `ai_output`：AI 第一次的原稿
- `user_edited`：你修正後的版本

### 2. 取得目前生效的提示詞

打開試算表 → `prompts` 分頁 → 找該分類 `active = TRUE` 那列 → 複製 `prompt` 格子內容。

### 3. 貼到高階模型，用下面的 meta-prompt 範本

把範本中兩個 `{{ }}` 區塊換成步驟 2、步驟 1 的內容：

````
你是一位提示詞工程專家，負責優化醫療文字整理系統的提示詞。
以下提供某分類「目前使用的提示詞」，以及多筆「AI 依此提示詞產出的原稿」
與「醫師實際修正後版本」的對照（JSON 格式，ai_output 為原稿、user_edited 為修正版）。

請分析醫師反覆修改的模式（格式、用詞、保留或刪除的內容、結構等），
找出 AI 系統性的不足，提出改進後的提示詞。

要求：
1. 只在必要處修改，保留原提示詞的整體結構與意圖
2. 著重「反覆出現」的模式，不要過擬合到個別案例
3. 先輸出「分析」，再輸出「可直接使用的新提示詞全文」（不要加引號或程式碼框）

【目前的提示詞】
{{貼上步驟 2 複製的提示詞}}

【編輯對照 JSON】
{{貼上步驟 1 匯出的 feedback JSON}}
````

### 4. 把新提示詞套用回去（手動改 `prompts` 表）

拿到滿意的新提示詞後，到試算表 `prompts` 分頁：

1. 複製該分類**目前那一列**，貼成新的一列
2. 新列：`version` +1、`prompt` 換成新版內容、`active` 設 `TRUE`、`note` 寫個來源備註
3. 舊列：`active` 改成 `FALSE`

存檔後，下次送出該分類就會用新提示詞。**系統規則：同分類取 `active=TRUE` 中 version 最大的那筆。**

> 想回溯：把舊列 `active` 改回 `TRUE`、新列改 `FALSE` 即可。所有版本都留著，不會遺失。

---

## 為什麼套用不能用網址一鍵？

`apply_prompt`（套用）是 POST 請求，瀏覽器網址列只能發 GET，所以手動流程的「套用」
就是直接編輯 `prompts` 表 —— 這正是當初把提示詞外部化到 sheet 的用意：改一格就生效。

---

## 兩種優化方式怎麼選

| | 🧬 一鍵優化（桌面端） | 手動拉出來跑（本文件） |
|---|---|---|
| 模型 | 你 GAS 設定的那個 | 任何你想用的高階模型 |
| 操作 | 點按鈕，數秒完成 | 多幾個複製貼上步驟 |
| 費用 | 走你的 API Key | 可用網頁版訂閱額度 |
| 適合 | 日常、快速迭代 | 重要分類、想要最高品質時 |

兩者並存：平常用 🧬，遇到要精雕的分類再用手動流程搭最強模型。
